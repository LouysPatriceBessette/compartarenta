# Screenshot plan (MVP)

This is a **plan** for screenshots so we can produce store‑ready assets later with minimal scrambling.

## Screens to capture (initial)

1. Home
2. Settings (shows version/build + privacy policy link)
3. Offline banner visible
4. Error state screen (fallback UI) — only if it can be triggered deterministically

## Devices / sizes (later)

We will produce at least:
- Android phone screenshots (Play)
- iPhone screenshots (App Store)

If we expand to tablets later, add:
- 7"+ Android tablet (optional)
- iPad (optional)

## How to keep screenshots consistent

- Keep a deterministic “demo mode” dataset (no personal data).
- Use the same locale per screenshot set (start with `en-US`).
- Keep status bar consistent (time/battery) if the store requires it (especially iOS).
